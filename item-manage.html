Item Management 2.0

Revamp the current Item Management System to cater to the needs of each department.
Item Management System 2.0 includes advanced features such as real-time tracking, automated reporting, and integration with other systems to provide a more comprehensive and efficient solution for managing items.

Problem Statement:

The current item management system lacks a comprehensive solution, making it difficult for departments to efficiently retrieve relevant data, highlighting the need for a redesigned system to provide centralized data access.

Redesigning the Item Management System is necessary to create a centralized repository with an intuitive interface, robust search function, and seamless integration to enable data easy access.

Challenges:

  • The current UI is simple and on a single page.
  • Item's creating and updating are handled by 1 department.
  • The rest departments only can use the data and calculate by themselves.

(Item Management 1.0 Information Architecture)

Goals:

  • Customization: Tailor the system to meet the specific requirements of each department.
  • Keep it concise: Simplify the new system to one page.
  • Collaboration: Involve all departments in the process of creating and updating the data.
  • Search function: Enhance the search function to meet varying needs.

My Role:

Product Designer (Research | Information Architecture | User Flow | Wireframe | Prototype | UI)

Discover

research and findings

When an organization has multiple departments, it is inevitable that the diverse needs of each department will present unique challenges. To effectively address these challenges, it is important to adopt a structured and methodical approach.

1. Gather information

In order to gather information for the design process, we collected relevant documents from each department and selected those that were most pertinent to the item in question.

  • Talk to colleagues directly
  • Arrange meetings
  • Send questionnaire emails
  • Search through departments' website

2. Organize & Analysis

Categorize the data into relevant groups and extract key insights to inform the design decisions. It's important to involve relevant stakeholders in this process to ensure the analysis is comprehensive and accurate.

3. Communications

I conducted research, prepared documents and arranged meetings with each department to understand their needs and clarify any questions I had. Some meetings were successful and others required follow-up, but I was able to begin the design process with the gathered information.

QUESTIONS

I analyzed the materials for each department and compiled a list of questions.

MEETINGS

I ask questions, take notes, and clarify requirements to better understand the needs.

Questionnaire

In cases where individuals are unable to attend a meeting, I create a questionnaire to gather their responses and input.

Design

user flow, wireframes, design system and UI

Information Architecture Rebuilt

All the basic modules have been kept, with additional common attributes and departments modules included. For each department module, a filter has been added to enhance the user experience and allow for more efficient navigation. Additionally, the modules have been optimized to ensure that the user can access the information they need quickly and easily.

User Flow

In order to obtain a comprehensive and thorough comprehension of the requirements for the application, I created basic task flows that outlined the step-by-step processes a user would realistically go through to achieve their intended objectives. This exercise provided clarity regarding the user's journey within the system.
